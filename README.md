## 🍳 프로젝트 소개

**15분식탁**은  
자취생과 1인 가구를 위한  
**15분 이내 · 저비용 레시피 공유 서비스**입니다.

복잡한 조리 과정과 긴 요리 시간을 줄이고,  
누구나 부담 없이 집밥을 즐길 수 있도록  
레시피 공유와 커뮤니티 기능을 중심으로 설계되었습니다.

### ✨ 핵심 컨셉
- ⏱️ 15분 이내 조리 기준
- 💰 저비용 재료 중심
- 🍽️ 자취생 맞춤 집밥
- 💬 레시피 중심 커뮤니티

## 🤝 협업 방식

#### ❇️**역할 분리 기반 협업**
 - 본 프로젝트는 프론트엔드와 백엔드를 분리하여 개발을 하였으나 백안에서 분담해서 다같이 만들며 프론트안에서 분담을 나눠 팀원들고 협업.<br>
 - 프론트엔드는 사용자 화면(UI/UX), 관리자 페이지, 대시보드 시각화를 담당. <br>
 - 백엔드는 API 설계, 데이터 처리, 관리자 통계 및 비즈니스 로직을 담당.<br>
 - 기능 구현 시 화면 요구사항을 기준으로 API 범위를 정의한 후 각 영역에서 병렬로 개발. <br>

## 🔍 브랜치 전략

#### ❇️ Pull Request(PR) 및 코드 리뷰

- **PR 생성**
  - 기능 개발 또는 버그 수정 작업이 완료되면 `main` 브랜치로 Pull Request를 생성한다.
  - PR은 반드시 해당 작업의 이슈와 연결한다.

- **PR 제목 규칙**
  - 작업 유형과 이슈 번호를 포함하여 작성한다.

- **PR 템플릿 사용**
  - PR 생성 시 공통 템플릿을 사용하여 아래 내용을 명확히 작성한다.
    - 작업 요약
    - 변경된 주요 코드 또는 파일
    - 해결된 이슈 내용
    - 테스트 여부 및 확인 사항

- **코드 리뷰**
  - 최소 **2명의 팀원**이 PR을 리뷰하고 승인한다.
  - 리뷰 시 다음 항목을 중점적으로 확인한다.
    - 코드 가독성 및 구조
    - 프론트엔드 ↔ 백엔드 연동 적합성
    - 불필요한 코드 또는 중복 로직 여부

- **PR Merge 및 이슈 완료**
  - 모든 리뷰 승인 후 PR을 `main` 브랜치에 병합한다.
  - PR이 merge되면 연결된 이슈는 자동으로 `Done` 상태로 처리한다.

#### ❇️**정기 회의 및 코드 리뷰**
 - **월요일**: 팀 회의를 통해 진행 상황 공유 및 문제 논의 & 주일간 예정 사항 정리<br>
 - **금요일**: 팀 회의를 통해 진행 상황 공유 및 문제 논의<br>
 - 프론트엔드는 UI를 시연, 백엔드는 API 및 기능 구현 리뷰.<br>
 - **스타일 및 규칙 정리**: 프론트엔드 컴포넌트 구조와 백엔드 레이어 구조를 유지하도록합의 반복되는 문제는 규칙으로 정리하어 이후 작업 반영
